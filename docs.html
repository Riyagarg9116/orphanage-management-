Project Overview
What this system does and why it was built

Objective

The Orphanage Management System (OMS) is a database management project designed to digitize and streamline the operations of an orphanage. The system replaces manual record-keeping with a structured relational database that stores and manages information about children, staff, donors, donations, and adoption processes.

The project demonstrates core DBMS concepts including entity-relationship modeling, normalization, SQL DDL/DML operations, JOIN queries, aggregate functions, and referential integrity through foreign keys.

Problem Statement

Orphanages typically manage large amounts of sensitive data manually — paper registers for children, handwritten donation receipts, and untracked adoption records. This leads to:

  • Loss of important records
  • Difficulty tracking adoption status
  • No visibility into donation history
  • Inability to generate reports quickly
  • No centralized staff management

This system solves all of the above with a structured relational database and a user-friendly interface.

Children

Track every child's name, age, gender, admission date and health status

Adoptions

Manage adoption requests, link them to children and track approval status

Donors

Maintain a registry of donors with contact information and donation history

Donations

Record every financial contribution with amount, date and donor reference

Staff

Manage all staff members — their roles, salaries and contact details

Dashboard & Reports

Visual dashboard with stats, monthly donation chart and recent activity feed

DDL — Create Tables

Create Database & Tables
-- Create and select the database
CREATE DATABASE IF NOT EXISTS orphanage_db;
USE orphanage_db;

-- Table 1: child
CREATE TABLE IF NOT EXISTS child (
    child_id       INT          NOT NULL,
    name           VARCHAR(50)  DEFAULT NULL,
    age            INT          DEFAULT NULL,
    gender         VARCHAR(10)  DEFAULT NULL,
    admission_date DATE         DEFAULT NULL,
    health_status  VARCHAR(30)  DEFAULT NULL,
    PRIMARY KEY (child_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- Table 2: adoption
CREATE TABLE IF NOT EXISTS adoption (
    adoption_id  INT         NOT NULL,
    child_id     INT         DEFAULT NULL,
    adopter_name VARCHAR(50) DEFAULT NULL,
    status       VARCHAR(20) DEFAULT NULL,
    PRIMARY KEY (adoption_id),
    FOREIGN KEY (child_id) REFERENCES child(child_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- Table 3: donor
CREATE TABLE IF NOT EXISTS donor (
    donor_id INT         NOT NULL,
    name     VARCHAR(50) DEFAULT NULL,
    contact  VARCHAR(15) DEFAULT NULL,
    PRIMARY KEY (donor_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- Table 4: donation
CREATE TABLE IF NOT EXISTS donation (
    donation_id INT  NOT NULL,
    donor_id    INT  DEFAULT NULL,
    amount      INT  DEFAULT NULL,
    date        DATE DEFAULT NULL,
    PRIMARY KEY (donation_id),
    FOREIGN KEY (donor_id) REFERENCES donor(donor_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- Table 5: staff
CREATE TABLE IF NOT EXISTS staff (
    staff_id INT         NOT NULL,
    name     VARCHAR(50) DEFAULT NULL,
    role     VARCHAR(50) DEFAULT NULL,
    salary   INT         DEFAULT NULL,
    contact  VARCHAR(15) DEFAULT NULL,
    PRIMARY KEY (staff_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

SELECT, JOIN & Aggregate Queries

Basic SELECT Queries
-- All children ordered by age
SELECT * FROM child ORDER BY age ASC;

-- Female children only
SELECT * FROM child
WHERE gender = 'Female';

-- Pending adoptions
SELECT * FROM adoption
WHERE status = 'Pending';

-- Staff with salary above 30000
SELECT * FROM staff
WHERE salary > 30000
ORDER BY salary DESC;

-- Donations sorted by date
SELECT * FROM donation
ORDER BY date DESC;
JOIN Queries
-- Adoption with child details
SELECT a.adoption_id, c.name, c.age,
       a.adopter_name, a.status
FROM adoption a
INNER JOIN child c
ON a.child_id = c.child_id;

-- Donation with donor name
SELECT d.donation_id, dn.name,
       d.amount, d.date
FROM donation d
INNER JOIN donor dn
ON d.donor_id = dn.donor_id;

-- Children NOT yet adopted
SELECT c.name, c.age, c.gender
FROM child c
LEFT JOIN adoption a
ON c.child_id = a.child_id
WHERE a.adoption_id IS NULL;
Aggregate Queries
-- Total children count
SELECT COUNT(*) AS total_children
FROM child;

-- Average age of children
SELECT AVG(age) AS avg_age FROM child;

-- Total donations received
SELECT SUM(amount) AS total
FROM donation;

-- Count by gender
SELECT gender, COUNT(*) AS count
FROM child GROUP BY gender;

-- Monthly donation totals
SELECT MONTH(date) AS month,
       SUM(amount) AS total
FROM donation
GROUP BY MONTH(date);
UPDATE & DELETE
-- Update child health status
UPDATE child
SET health_status = 'Good'
WHERE child_id = 4;

-- Approve an adoption
UPDATE adoption
SET status = 'Approved'
WHERE adoption_id = 2;

-- Update staff salary
UPDATE staff
SET salary = 65000
WHERE staff_id = 3;

-- Delete rejected adoptions
DELETE FROM adoption
WHERE status = 'Rejected';
System Features
Everything the Orphanage Management System can do

Frontend Features

  • Dashboard — Live stats cards, monthly donation bar chart, recent activity feed
  • Children Management — Add, edit, delete children with health status badges
  • Adoption Tracking — Create adoption records, toggle status (Pending/Approved/Rejected)
  • Staff Management — Full CRUD for staff with role and salary tracking
  • Donor Registry — Manage donors, view total donated per donor
  • Donation Records — Log donations linked to donors with date and amount
  • Search & Filter — Real-time search on every table, filter by gender/status
  • SQL Viewer — All queries with syntax highlighting and copy button
  • Local Storage — Data persists across page refreshes in the browser
  • Toast Notifications — Feedback on every action (add, edit, delete)
  • Responsive Design — Works on desktop, tablet and mobile

Database Features

  • 5 Normalized Tables — child, adoption, donor, donation, staff
  • Primary Keys — Every table has a unique identifier
  • Foreign Keys — adoption.child_id → child, donation.donor_id → donor
  • Referential Integrity — InnoDB engine enforces FK constraints
  • DDL Queries — CREATE DATABASE, CREATE TABLE with constraints
  • DML Queries — INSERT, UPDATE, DELETE operations
  • SELECT Queries — 12+ queries with WHERE, ORDER BY, LIKE
  • JOIN Queries — INNER JOIN and LEFT JOIN across related tables
  • Aggregate Functions — COUNT, SUM, AVG, MAX, MIN, GROUP BY, HAVING
  • Subqueries — Children not yet adopted using nested SELECT

How to Use the System
1

Open the Application

Open index.html in any web browser. The dashboard loads automatically with sample data.

2

Navigate Using the Sidebar

Click any section in the left sidebar — Children, Adoptions, Staff, Donors, Donations, or SQL Queries.

3

Add New Records

Click the green "Add" button on any page. Fill in the form fields and click Save. The record appears instantly in the table.

4

Edit or Delete Records

Each row has an Edit (pencil) and Delete (trash) button. Edits open the same form pre-filled. Deletes ask for confirmation first.

5

Search and Filter

Use the search box at the top of each table to filter records in real time. Use the dropdown to filter by gender or adoption status.

6

View SQL Queries

Click SQL Queries in the sidebar to see all database queries organized by type — DDL, DML, SELECT, JOIN, UPDATE/DELETE, and Aggregate.
